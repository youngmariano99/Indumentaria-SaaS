# Ticker: PWA - Ergonomía Móvil y UX Adaptativa

**Fecha y Hora:** 2026-03-04 19:55
**Tipo de Cambio:** Feature UX/UI

## Impacto Visual
El software ahora detecta inteligentemente si está siendo operado desde una pantalla chica (celulares/tablets verticales) o desde un monitor de escritorio, mutando drásticamente su forma de navegación para privilegiar la "regla del pulgar" operativa.

## Detalle Técnico
1. **Navegación Móvil Bottom-Bar (`MobileTabBar.tsx`)**: Se extrajo la navegación lateral pesada hacia un componente de tipo "Tab Bar" fijo en la parte inferior para celulares. Contiene atajos a Inicio, Stock, POS (destacado redondo) y CRM. El botón de "Más" dispara un evento que aprovecha la Sidebar original para usarla como "Drawer" lateral.
2. **Visual Viewport API (`useVisualViewport.ts`)**: Se creó un Custom Hook para detectar la aparición del teclado nativo (Soft-keyboard) validando diferencias de `innerHeight`. Esto previene fallos comunes de CSS donde los botones fijos en *fixed bottom* quedan ocultando lo que el usuario está tipeando.
3. **Modo Oscuro Autoadaptativo (`variables.css`)**: Se incluyó una query `@media (prefers-color-scheme: dark)` que *sobreescribe* en caliente la escala de grises de Tailwind/Custom (`--color-gray-50` hasta `--color-gray-900`) e invierte la paleta principal hacia tonos sin intensidad destellante. Como todos los componentes del sistema (como tarjetas, inputs) consumen estas variables, todo el SaaS se transforma instantáneamente a diseño Dark sin programar un componente switcher manual.

## Explicación Didáctica
Si abrimos tu SaaS desde el celular, ya no vamos a ver una pantallita comprimida ni un menú incómodo escondido arriba. Ahora nos encontramos con esto:
1. **La barra de Instagram:** Literalmente construí un menú inferior igual al de las grandes redes sociales. Si el vendedor tiene el celular en una mano escaneando ropa, el pulgar le llega perfectamente al botón central gigante del "CARRITO" para cobrar.
2. **Inteligencia con el Teclado:** Un gran dolor de cabeza en el desarrollo web es que cuando tocás un campo de texto (ej. "Buscar Cliente"), el celular saca su teclado y achica toda la página rompiendo el diseño. Yo metí un "detector" (`VisualViewport`) que le avisa a React "guarda que están escribiendo". Entonces, la barra inferior desaparece sutilmente para darle todo el alto de la pantalla a la búsqueda.
3. **Mimos para la vista (Modo Noche):** Modificando los códigos de color en el corazón del proyecto, programé que si el empleado tiene su celular configurado en "Modo Oscuro", todo tu sistema (fondos, botones, letras) se invoca con fondo negro azabache y grises mate, ideal para no encandilar de noche en un depósito u oficina o en un punto de venta y que se vea increíblemente Premium.
